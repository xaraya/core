1. Sanity Tests


1.1
 Command line arguments

--verbose enables verbose output

--all runs all checks

no parameters runs only default checks

--checks=0.0.1 runs only check 0.0.1

--checks=0.0.1,0.0.2 runs both check 0.0.1 and 0.0.2

Program can operates on multiple files.

Executing with no command line parameters displays the help.

--weave runs each test on every file

--noweave runs every test on each file
1.2
 Checks

Each normal check requested loaded and executed.

Each regexp check requested loaded and executed correctly.

Checks with 'enabled' = false will not run.

Check with 'filetype' = template will only run on templates.

Check with 'filetype' = all will run on all files.

Failing a check with 'fatal' = false does not exit with an error code.
$ bk qachecks2 --checks=0.0.1 <file-larger-than-10K> || echo "You shouldn't see this"

Failing a check with 'fatal' = true does exit with an error code
$ bk qachecks2 --checks=0.0.2 <file-with-the-text-'Xaraya'> || echo "Should see this"
